# 🤖 Project: Persona-AI Clicker (Web-based MVP)

본 프로젝트는 초등학생을 대상으로 한 **생성형 AI 기반 교육용 대화 웹 서비스**입니다. 하드웨어 비용을 최소화하고 접근성을 높이기 위해 스마트폰/노트북의 기능을 활용하며, 성인(보호자/교사)의 계정 인증을 통해 미성년자 사용 제한 문제를 해결합니다.

## 🌟 핵심 컨셉

* **Thin Client:** 하드웨어(클릭커/피규어)는 마이크와 스피커 역할만 수행하고, 복잡한 연산과 AI 처리는 연결된 기기(핸드폰/PC)가 담당합니다.
* **Safety First:** 관리자가 설정한 '마스터 프롬프트'를 기반으로 유해 콘텐츠를 원천 차단합니다.
* **Easy Access:** 복잡한 API 키 발급 없이 성인 계정 로그인(OAuth)만으로 즉시 사용 가능합니다.

---

## 🛠 시스템 아키텍처

1. **사용자 단말기 (Web):** 반응형 웹을 통해 PC, 노트북, 스마트폰에서 접속.
2. **입력 (STT):** 단말기의 마이크를 통해 학생의 음성을 텍스트로 변환.
3. **처리 (Gemini API):**
* **Master Prompt:** 개발자/관리자가 고정한 안전 및 교육 가이드라인.
* **User Persona:** 학생이 선택/설정한 대화 상대의 성격(캐릭터, 성별 등).


4. **출력 (TTS):** Gemini의 스트리밍 답변을 음성으로 변환하여 스피커로 송출.

---

## 📋 주요 기능

### 1. 마스터 및 유저 페르소나 설정

* **Master Control:** 미성년자 보호를 위한 엄격한 윤리 가이드라인 고정 (수정 불가).
* **User Customization:** * 제시된 템플릿(다정한 선생님, 로봇 친구 등) 선택.
* 학생이 직접 대화할 친구의 페르소나(직업, 성격, 호칭) 생성.



### 2. 지연 시간(Latency) 최적화 UX

* **Streaming Response:** 답변 전체가 완성되기 전 문장 단위로 즉시 읽어주기.
* **Filler Sound:** AI가 생각하는 동안 "음~", "잠시만요!" 등 효과음을 삽입하여 체감 대기 시간 단축.

### 3. 법적/윤리적 보호 장치

* **Proxy Authentication:** 부모/교사 계정을 통한 대리 인증으로 미성년자 직접 가입 문제 해결.
* **Safety Filtering:** 입력 및 출력 단계에서 2중 필터링 시스템 가동.
